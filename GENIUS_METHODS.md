## Badge-by-Badge Guide

### 🦈 Pull Shark

**Goal:** Get merged PRs in public repositories.

**Genius Approach:**
1. Find beginner-friendly open-source projects
2. Fix real bugs, improve documentation, add tests
3. Collaborate with maintainers
4. Get PRs merged naturally

**Timeline:** 
- Bronze (4 PRs): 1 month
- Silver (16 PRs): 3 months  
- Gold (128 PRs): 12+ months

**Skill Gain:** ⭐⭐⭐⭐⭐
- Learn git workflows
- Code review processes
- Collaboration
- Multiple codebases
- Problem-solving

**Resources:**
- See `/badges/pull-shark/GENIUS.md`
- https://firsttimersonly.com
- https://goodfirstissue.dev

### 🌟 Starstruck

**Genius Approach:**

**Timeline:**
- Bronze (16 stars): 2-8 weeks
- Silver (128 stars): 3-6 months
- Gold (512 stars): 1-2+ years

**Skill Gain:** ⭐⭐⭐⭐⭐
- Product thinking
- Marketing basics
- Maintaining open source
- Building in public
- Community building

**Resources:**

- ProductHunt tips

**Examples:**
- Simple CLI tool → 50 stars in 2 months

### 🌠 Galaxy Brain

**Goal:** Get answers marked as "accepted" in GitHub Discussions.

**Genius Approach:**
1. Find active repositories with Discussions
2. Read open questions
3. Research & provide helpful, detailed answers
4. Let maintainers mark your answer as accepted

**Timeline:**
- Bronze (1 answer): 1 week
- Silver (10 answers): 2 months
- Gold (25 answers): 6 months

**Skill Gain:** ⭐⭐⭐⭐
- Communication
- Mentoring
- Product knowledge
- Problem-solving
- Helping others

**Best Repositories for Answers:**
- `vercel/next.js` — Very active discussions
- `facebook/react` — Tons of questions
- `nodejs/node` — Great community
- `python/cpython` — Always questions
- Any popular framework

**Tips:**
- Answer within 24 hours (more likely to be marked)
- Provide detailed, well-formatted answers
- Include code examples
- Test your solution before answering
- Be kind and patient

### 💬 Pair Extraordinaire

**Goal:** Co-author commits on merged PRs.

**Genius Approach:**
1. Collaborate with teammates/friends on real projects
2. Use git co-author syntax
3. Submit PR with multiple co-authors
4. Get PR merged

**Timeline:**
- Bronze (1 co-authored): Immediate
- Silver (10 co-authored): 2 months
- Gold (24 co-authored): 6 months

**Skill Gain:** ⭐⭐⭐
- Pair programming
- Collaboration
- Code review
- Communication

**How to Co-author:**
```bash
# In commit message, add:
git commit -m "Implement feature X

Co-authored-by: Friend Name <friend@example.com>
Co-authored-by: Another Dev <another@example.com>"
```

**Best Practices:**
- Actual collaboration (not fake)
- Both people should contribute meaningfully
- Use for real development work
- Pair programming sessions

### 💝 Sponsor

**Goal:** Sponsor a developer or organization.

**Genius Approach:**
1. Find an open-source maintainer you admire
2. Visit their GitHub Sponsors page
3. Contribute $1-$10/month
4. Badge earned instantly

**Timeline:** Immediate!

**Skill Gain:** ⭐
- Support open source
- Financial contribution
- Show gratitude to maintainers

**How to:**
- Visit https://github.com/sponsors/USERNAME
- Click "Sponsor"
- Choose tier
- Get badge!

**Cost:** $1-5/month is enough for the badge

### ⚡ Quickdraw

**Goal:** Close a PR or issue within 5 minutes.

**Genius Approach:**
1. Create PR in your own repo
2. Merge immediately (within 5 minutes)
3. Badge earned!

**Timeline:** Instant (requires nothing else)

**Skill Gain:** ⭐
- Just fun!
- Not much learning value

**Realistic Use Case:**
```bash
# Fixing urgent bug
git checkout -b hotfix/security-issue
# ... fix bug ...
git push
gh pr create --title "Security: Fix XSS vulnerability"
gh pr merge <number> --merge  # Merge within 5 min!
# Badge earned!
```

---

### 🦄 YOLO

**Goal:** Merge a PR without review.

**Genius Approach:**
1. Create PR in your own repo
2. Merge without waiting for reviews
3. Badge earned!

**Timeline:** Instant

**Skill Gain:** ⭐
- Nothing (it's just for fun)

**When It Makes Sense:**
- Experimental branch
- Testing new feature
- Personal project
- Documentation updates

**When NOT to Use:**
- Team/work projects
- Shared codebases
- Production code
- When reviews are needed
