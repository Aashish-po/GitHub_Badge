# 🛡️ Security Advisory Credit Highlight - SVG Assets

## About the Highlight

The **Security Advisory Credit** highlight recognizes contributors who have helped identify and fix security vulnerabilities.

### Highlight Overview

- **Name:** Security Advisory Credit
- **Type:** Security Achievement
- **Status:** Active/Available
- **Requirement:** Contributing to security advisories
- **Mode Variants:** Light Mode and Dark Mode

## What is a Security Advisory Credit?

When you help identify and remediate a security vulnerability, you can receive credit in a GitHub Security Advisory.

### Types of Contributions

🔍 **Finding Vulnerabilities:**
- Discover security issues
- Report through responsible disclosure
- Provide proof of concept (PoC)

🔧 **Fixing Vulnerabilities:**
- Develop patches
- Submit pull requests
- Review security changes

📝 **Advisory Participation:**
- Provide expert analysis
- Validate fixes
- Documentation contributions
- Testing and verification

## GitHub Security Advisory Process

### Step-by-Step

1. **Vulnerability Discovered**
   - Researcher or maintainer identifies issue
   - Report through security form

2. **Advisory Created**
   - GitHub creates private advisory
   - Collaborators are invited
   - Solution is developed

3. **Patch Development**
   - Team works on fix
   - Testing and verification
   - Public disclosure timeline

4. **Credit Assignment**
   - Contributors are credited
   - Badge earned on profile
   - Public CVE published
